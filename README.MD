## Multi-Screen Navigation â€” Sprint 3.16

DormDesk uses Flutterâ€™s Navigator and named routes to enable navigation between Welcome, Login, Signup, and Home screens.

### Routes Setup

Routes are defined in `main.dart`:

```dart
MaterialApp(
  initialRoute: '/welcome',
  routes: {
    '/welcome': (context) => WelcomeScreen(),
    '/login': (context) => LoginScreen(),
    '/signup': (context) => SignupScreen(),
  },
  onGenerateRoute: (settings) {
    if (settings.name == '/home') {
      final uid = settings.arguments as String;
      return MaterialPageRoute(
        builder: (context) => HomeScreen(uid),
      );
    }
    return null;
  },
);
```

### Navigation Flow

```
WelcomeScreen â†’ LoginScreen â†’ HomeScreen
SignupScreen â†’ HomeScreen
Logout â†’ LoginScreen
```

### Navigator Methods Used

* `Navigator.pushNamed()` â€“ Navigate to another screen
* `Navigator.pushReplacementNamed()` â€“ Replace current screen
* `Navigator.pop()` â€“ Go back
* `onGenerateRoute` â€“ Pass UID to HomeScreen

### Argument Passing Example

```dart
Navigator.pushReplacementNamed(
  context,
  '/home',
  arguments: user.uid,
);
```

### Reflection

Using named routes makes navigation clean, scalable, and easier to manage in larger applications like DormDesk. It also allows safe data transfer between screens.

This feature demonstrates scrollable layouts in DormDesk using Flutterâ€™s ListView and GridView widgets. These widgets allow efficient display of dynamic and large datasets while maintaining smooth UI performance.

---

### ListView Implementation

ListView.builder is used to display a horizontal scrollable list of hostel-related items.

```dart

  height: 200,
```
